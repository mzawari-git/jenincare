# SkinAnalyzer — Comprehensive Upgrade & Architecture Plan
## خطة التطوير الشاملة لنظام تحليل البشرة

> **Current Status (June 2026)**: The system has a fully functional Laravel backend (Sanctum API, 4 AI providers, queue jobs, multi-channel image support, e-commerce, Meta ads integration) and a pre-compiled Android APK (USB camera, TensorFlow Lite, OpenCV). The Laravel backend source is in the repo; the Android APK binary and its decompiled smali are in `SkinAnalyzer/`. No Android/Kotlin source code is in the repo — the APK is built externally.
>
> **Goal**: Evolve from "a working app" to "a specialized medical-grade dermatology station."

## Track 1: Camera Hardware Control (AE/AWB Lock + Burst)
## المسار 1: السيطرة الكاملة على الكاميرا

### Status: ❌ MISSING (Android — depends on source rebuild decision)

### Goal
Lock AE/AWB/AF parameters and execute burst capture across 3 light channels in sub-200ms intervals.

### Current Reality
- The existing APK uses **USB/UVC external camera** via `libUVCCamera.so` (com.serenegiant), not CameraX/Camera2.
- TensorFlow Lite + OpenCV are bundled for on-device inference.
- The repository has **no Android source code** — only decompiled smali.
- `scan_analysis_images` table already supports 4 modes: `rgb`, `cross`, `parallel`, `uv`.

### If Android is rebuilt from source (Kotlin + CameraX):

#### [NEW] `CameraControlManager.kt`
```
app/src/main/java/com/jenincare/skinanalyzer/
└── camera/
    ├── CameraControlManager.kt       ← AE/AWB/AF lock via Camera2Interop
    ├── BurstCaptureOrchestrator.kt   ← Sequential burst <200ms per shot
    ├── LightingChannel.kt            ← enum: RGB, UV, CROSS_POLARIZED
    └── CameraCalibrationData.kt      ← data class: ISO, WB, exposure, ambient
```

**Logic — `CameraControlManager.kt`:**
```kotlin
fun lockCameraParameters(cameraControl: CameraControl): Flow<LockState> {
    // 1. Lock AE (Auto Exposure)
    // 2. Lock AWB (Auto White Balance) via Camera2Interop
    // 3. Lock AF (Auto Focus) after initial focus
    // 4. Log ISO + Exposure Time as CalibrationData
}
```

**Logic — `BurstCaptureOrchestrator.kt`:**
```kotlin
suspend fun captureBurst(): BurstResult {
    // Channel 1: RGB (standard illumination)
    // Channel 2: UV (violet filter — currently simulated via color filter)
    // Channel 3: Cross-Polarized (simulated via contrast/saturation)
    // Interval between captures: < 200ms
    // Post-capture: Image alignment check
}
```

### USB Camera Support (com.serenegiant UVCCamera)
If keeping the existing USB camera approach:
- Wrap `libUVCCamera` in a Kotlin/Compose compatible layer
- Add AE/AWB control via UVCCamera's native controls (if supported by hardware)
- Maintain backward compatibility with existing USB cameras

> [!TIP]
> Phase 1 (Mock Protocol): Use 3 static test images from `assets/test_images/` to test the full pipeline end-to-end before hardware integration.

public function addRecommendedProducts(int $scanId, int $userId): bool {
    $scan = SkinScan::with('recommendedProducts')->findOrFail($scanId);
    $cart = Cart::firstOrCreate(['user_id' => $userId]);
    foreach ($scan->recommendedProducts as $product) {
        $cart->items()->updateOrCreate(

            ['quantity' => 1]
        );
    }

}
